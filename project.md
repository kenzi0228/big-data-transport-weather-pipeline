# Big Data Ecosystem project

## Objective:

The objective of this project is to introduce you to the fundamental concepts of Big Data through the design of a simple but coherent data pipeline, based on the Hadoop ecosystem studied in class.

This project does not focus on performance or technical complexity, but rather on your ability to:

- understand the role of each component,
- describe an end-to-end data flow,
- justify your choices in your own words.
- Processing can be performed in batch, streaming, or hybrid mode.

## Deadline

Monday, 13 April 2026

## Instructions:

### 1. Context and use case (simple and concrete)

Describe the problem to be solved using data.

The use case must be realistic, easy to understand, and consistent with Big Data processing (volume, frequency, or diversity of data).

Expected examples (for guidance):
- sensor data analysis,
- event tracking,
- log analysis,
- transportation, weather, social media data, etc.

### 2. Data: origin and structure

- Describe the origin of the data: public dataset or data generated by yourself.
- Present the data structure: format (structured, semi-structured, unstructured), schema or field description
- estimated volume and frequency of data arrival (in the case of streaming).

### 3. Your Big Data architecture

Indicate whether your pipeline is based on batch, streaming, or hybrid processing.

Present the overall architecture of the pipeline, specifying the components used, their respective roles (without going into advanced technical details), and the interactions between them.

Describe the different stages of the pipeline, from data ingestion to the use of results.

The description should provide a clear understanding of the end-to-end data flow.

### 4. Testing and validation

Show your tests carried out on the different components of the pipeline.
